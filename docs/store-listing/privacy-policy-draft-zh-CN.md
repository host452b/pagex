# Pagex 隐私政策（中文）

最后更新：`2026-04-11`

## 概述

Pagex 是一个 Chrome 插件，用于将用户选中的网页标签页提取为结构化 JSON，供本地工作流使用。

本草案基于当前仓库中的实际行为编写：

- 页面解析在浏览器内完成
- 结果会存入扩展 session storage，用于状态恢复与摘要展示
- 只有在用户明确点击 `COPY.JSON` 后，结果才会写入剪贴板
- 插件默认不会把页面内容发送到远程服务器

## 我们可能处理的信息

当你使用 Pagex 时，插件可能处理：

- 目标页面的 URL 和标题
- 页面中的可见内容
- 可安全采集的隐藏内容和可展开内容
- DOM 结构、frame 元信息、经过过滤的属性和有限的样式摘要
- 扩展的 session 状态，例如解析状态、摘要信息和最近一次生成的 payload

## 我们不主动收集的信息

Pagex 的设计目标之一，是避免默认收集不必要的敏感数据。

当前实现中：

- 默认只保留部分属性
- 会对敏感属性名进行脱敏处理
- 会移除导出 `href` / `src` 中的 query 和 hash
- 默认不会把提取结果上传到远程后端

## 权限说明

Pagex 使用以下 Chrome 权限：

- `activeTab`
- `scripting`
- `storage`
- `tabs`
- `webNavigation`
- 可选站点权限，仅在用户选择解析某个站点 origin 时动态申请

这些权限用于：

- 获取目标 tab
- 注入页面采集脚本
- 检查 frame 覆盖情况
- 在 popup 中恢复 session 状态
- 按需请求目标站点访问权限

## 本地存储

Pagex 会把解析状态和结果摘要存入扩展的 session storage。

这些本地数据用于：

- 在重新打开 popup 时恢复状态
- 展示 frame 数量、结果大小、warning 等摘要
- 在用户主动点击复制前保留最近一次 payload

## 剪贴板

只有在用户明确点击 popup 中的复制操作后，Pagex 才会把数据写入剪贴板。

## 数据共享

Pagex 默认不会将提取到的页面数据共享给远程服务。

如果未来版本加入远程处理、分析服务或云同步能力，必须在发布前同步更新本隐私政策以及 Chrome Web Store 的隐私字段。

## 用户可控项

你可以：

- 自行选择需要解析的标签页
- 拒绝站点权限申请
- 不点击复制而仅查看状态
- 随时卸载扩展
